    Q.Exception vs Throwable

    In Java, both Error and Exception are subclasses of the Throwable class, but they represent different types of problems
     and are handled differently.
    Exceptions are generally recoverable problems that occur due to issues in the application code,
            such as invalid user input or network failures, and should be handled programmatically.
    Errors are serious, unrecoverable problems that occur outside the control of the application, usually within the
            Java Virtual Machine (JVM) or the system environment, and most applications should not try to catch them.

    Key Differences: Error vs. Exception

    Key Distinctions
    Error indicates a problem that is outside the scope of the application code's control, such as running out of memory
        or a fundamental issue with the Java Virtual Machine (JVM). Trying to catch and handle these generally does not
        make sense because the program is in an unstable state and should usually be allowed to terminate.
    Exception is intended for conditions that a reasonable application might want to anticipate and handle programmatically.
        For example, if a program tries to open a file that doesn't exist, a FileNotFoundException is thrown,
        which the programmer can catch and manage gracefully (e.g., by prompting the user for a new file path).
    =========================================================================================================================
    Checked vs. Unchecked Exceptions: All errors are unchecked. Exceptions, however, are further divided:
    Checked exceptions (e.g., IOException) must be handled explicitly in the code (using try-catch or a throws clause) for
        the program to compile.
    Unchecked exceptions (e.g., RuntimeException subclasses like NullPointerException or ArrayIndexOutOfBoundsException) are
        typically the result of programming errors and do not need to be explicitly handled by the compiler,
         although they can be caught.

   Q.5 what is exception propagation in java
        Exception propagation in Java is the mechanism by which an exception is passed up the method call stack from
        the method where it occurred to its calling methods, until it is caught and handled by an appropriate try-catch block
        or reaches the top of the stack and terminates the program.
